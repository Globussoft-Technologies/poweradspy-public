          ❔ How this tab works (click to expand)
          

            
1. Pick a plan from the left list, grouped by tier (color-coded). "(2026)" groups are the new pricing-restructure tiers — kept separate from the legacy same-named tiers so they never collide.

            
2. Click ✏️ Edit (top-right of the detail panel) to change which networks/filters/competitor limits that plan ID gets, then 💾 Save. Changes write to MongoDB (plan_access_config) immediately — no deploy needed.

            
NEW / REVIEW badges: when a brand-new filter is added via SDUI, it's auto-granted to the top-tier plans (Palladium) by default and flagged REVIEW. Click "Review →" in the yellow strip above to jump straight to the first plan/row that needs a decision — extend access down to lower tiers as appropriate, then it clears itself.

            
Full reference: docs/PLAN_ACCESS.md in the repo — architecture, the 2026 pricing restructure, and a step-by-step manual QA scenario for testing plan changes end-to-end.
